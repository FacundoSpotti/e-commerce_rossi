# Restricciones del obligatorio

El proyecto es una SPA en React para un obligatorio de facultad.

Debe centrarse en:
- interfaz de usuario;
- navegación;
- estructura visual;
- prototipo funcional.

No debe incluir:
- backend;
- API;
- base de datos;
- persistencia real de datos.

Los datos deben estar hardcodeados.

## Tecnologías permitidas

- React
- Vite
- JavaScript
- CSS
- React Router
- React Hook Form
- Joi
- @hookform/resolvers
- useState
- useEffect solo si es necesario
- props
- map
- eventos onClick y onSubmit

## Tecnologías prohibidas

- TypeScript
- Tailwind
- Bootstrap
- Material UI
- Chakra UI
- Ant Design
- Redux
- Zustand
- Context API
- Next.js
- Firebase
- APIs externas
- localStorage salvo autorización explícita
- bases de datos
- librerías de animación complejas

## Funcionalidades obligatorias

La app debe incluir:
- Login
- Registro
- Home
- Catálogo o listado de productos
- Detalle de producto
- Carrito
- Checkout / Finalizar compra
- Mensaje de compra confirmada
- Perfil de usuario
- Historial de compras hardcodeado
- Espacio visual reservado para mapa

La validación de formularios debe hacerse con React Hook Form.
Joi puede usarse para validaciones simples.
