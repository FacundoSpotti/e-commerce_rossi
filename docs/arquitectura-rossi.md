# Arquitectura recomendada para ROSSI

La web debe funcionar como ecommerce, pero con una experiencia serena y artesanal.

La prioridad no es mostrar descuentos, sino transmitir:
- oficio;
- calidad;
- funcionalidad;
- origen;
- uso cotidiano;
- durabilidad.

## Naming recomendado

Usar:
- Finalizar compra

En lugar de:
- Checkout

Usar:
- Compra confirmada

En lugar de:
- Mensaje

Usar:
- Guardados

En lugar de separar Favoritos y Lista de deseos.

## Header global

El header debe incluir:
- Logo ROSSI
- Tienda
- Bolsos grandes
- Bolsos medianos
- Bolsos chicos
- Mochilas
- Nuestro taller
- Contacto
- Mi cuenta
- Carrito

Para simplificar el obligatorio, no es necesario implementar búsqueda real ni páginas institucionales completas.

## Jerarquía principal de páginas

- Home
- Catálogo
- Categorías
- Detalle de producto
- Carrito
- Finalizar compra
- Compra confirmada
- Login
- Registro
- Perfil

## Perfil

El perfil debe incluir:
- Mis pedidos
- Guardados
- Mis datos

Para el obligatorio, estos datos pueden ser ficticios y hardcodeados.

## Mapa

Si el taller está en la casa de Verónica, no conviene mostrar una dirección exacta.

El espacio visual del mapa debe mostrarse como:
- zona aproximada;
- atención con cita previa;
- bloque visual reservado para mapa.

Texto sugerido:
Atención con cita previa en Montevideo, Uruguay.
